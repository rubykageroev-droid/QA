# Баг-репорты — Форма логина / Корзина

---

## BUG-001 — Форма логина не реагирует на клавишу Enter

| Поле | Значение |
|---|---|
| **ID** | BUG-001 |
| **Severity** | Major |
| **Priority** | High |
| **Статус** | Open |
| **Окружение** | Windows 11, Chrome 122, https://saucedemo.com |

**Предусловие:** Пользователь открыл страницу логина, поля не заполнены.

**Шаги воспроизведения:**
1. Открыть https://saucedemo.com
2. Ввести в поле Username: `standard_user`
3. Ввести в поле Password: `secret_sauce`
4. Нажать клавишу **Enter** (не кнопку LOGIN)

**Ожидаемый результат:** Форма отправляется, пользователь авторизуется и попадает на страницу `/inventory.html`

**Фактический результат:** Ничего не происходит. Форма не отправляется, страница не меняется.

**Почему это важно:** Пользователи привыкли отправлять формы через Enter. Отсутствие этого поведения — UX-дефект и нарушение стандартного поведения веб-форм.

## BUG-002 — Поле Email принимает невалидный формат (строку без @)

| Поле | Значение |
|---|---|
| **ID** | BUG-002 |
| **Severity** | Major |
| **Priority** | Medium |
| **Статус** | Open |
| **Окружение** | Windows 11, Chrome 122, форма логина |

**Предусловие:** Пользователь на странице логина.

**Шаги воспроизведения:**
1. Открыть форму логина
2. Ввести в поле Email: `abc` (строка без символа @)
3. Ввести любой пароль: `123456`
4. Нажать кнопку отправки формы

**Ожидаемый результат:** Форма не отправляется, отображается сообщение об ошибке: _"Введите корректный email адрес"_

**Фактический результат:** Форма принимает введённые данные и отправляется без валидации формата email.

**Почему это важно:** Отсутствие валидации на фронтенде позволяет отправлять мусорные данные на сервер. Увеличивает нагрузку на backend и создаёт потенциальную уязвимость.

## BUG-003 — Пустой пароль принимается формой без ошибки

| Поле | Значение |
|---|---|
| **ID** | BUG-003 |
| **Severity** | Critical |
| **Priority** | High |
| **Статус** | Open |
| **Окружение** | Windows 11, Chrome 122, форма логина |

**Предусловие:** Пользователь на странице логина.

**Шаги воспроизведения:**
1. Открыть форму логина
2. Ввести валидный email: `test@test.com`
3. Оставить поле Password **полностью пустым**
4. Нажать кнопку отправки формы

**Ожидаемый результат:** Форма не отправляется, отображается ошибка валидации: _"Пароль обязателен для заполнения"_

**Фактический результат:** Форма отправляется без сообщения об ошибке. Пустой пароль проходит валидацию на фронтенде.

**Почему это важно:** Критический дефект. Отсутствие обязательной валидации поля пароля — потенциальная угроза безопасности и нарушение базовой логики аутентификации.

## BUG-004 — Товар не удаляется из корзины при изменении количества на 0

| Поле | Значение |
|---|---|
| **ID** | BUG-004 |
| **Severity** | Major |
| **Priority** | Medium |
| **Статус** | Open |
| **Окружение** | Windows 11, Chrome 122, корзина |

**Предусловие:** Пользователь авторизован, в корзине есть минимум 1 товар с количеством 1.

**Шаги воспроизведения:**
1. Авторизоваться как `standard_user`
2. Добавить любой товар в корзину
3. Перейти в корзину
4. Изменить количество товара на `0`
5. Подтвердить изменение (если есть кнопка) или нажать Tab/Enter

**Ожидаемый результат:** Товар автоматически удаляется из корзины, счётчик корзины обнуляется.

**Фактический результат:** Товар остаётся в корзине с количеством 0. Пользователь может перейти к оформлению заказа с нулевым количеством товара.

**Почему это важно:** Нарушение бизнес-логики. Оформление заказа с 0 единиц товара может вызвать ошибки на стороне бэкенда и системы складского учёта.
